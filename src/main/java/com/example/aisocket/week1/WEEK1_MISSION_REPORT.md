## 1주차 정리


### **AI & Java 언어 딥다이브 스터디 개편 커리큘럼**

### ** 1주차 - 런타임 메커니즘 & 모델 추상화 설계**

- **목표:** JVM 내부 메커니즘과 자바의 객체 지향 설계를 딥다이브하고, 다중 AI LLM 프로바이더(OpenAI, Ollama 등)를 유연하게 스위칭할 수 있는 고성능 채팅 서버의 뼈대를 구축합니다.
- **선행학습**
    - JVM, JVM의 메모리 3대 영역
    - Call by Value
    - 객체 지향 다형성(Polymorphism)
- **주요 학습 개념**
    - 컴파일/인터프리터(JIT) 동작 원리, 객체 헤더(Object Header)와 자바 타입 시스템의 메모리 레이아웃
    - 톰캣(Tomcat)의 Acceptor/Worker 스레드 풀 구조 및 HTTP vs 정적 소켓 통신 패킷 구조 비교
    - AI 백엔드 설계를 위한 인터페이스 추상화 및 다형성 극대화 패턴
- **미션**
    - **[미션: 코딩테스트] 대용량 AI 프롬프트 토큰(Token) 분할 및 슬라이딩 윈도우 알고리즘**
        - **조건:** LLM의 콘텍스트 윈도우 한계를 극복하기 위해, 대량의 채팅 로그 스트림을 받아 글자 수가 아닌 '토큰 수' 기준(간이 인코더 구현)으로 메모리 오버헤드 없이 실시간 큐를 관리하는 `O(N)` **최적화 슬라이딩 윈도우 알고리즘** 구현.
    - **[미션 1: 로레벨 검증] Wrapper 객체 오버헤드 프로파일링 & 소켓 버퍼 벤치마크**
        - **조건:** 수백만 개의 AI 대화 임베딩 데이터(실수형 벡터)를 자바 `Double[]` 객체 배열과 프리미티브 `double[]` 배열에 적재했을 때의 물리적 힙 메모리 점유율을 `VisualVM`으로 측정하고, **소켓 전송 시 CPU 캐시 미스 비용 비교 분석**하기
    - **[미션 2: 실무 구현] 다중 AI Engine 프로바이더 추상화 기반 `단일 스레드` 소켓 채팅 서버**
        - **조건:** 클라이언트가 소켓으로 메시지를 보내면 외부 AI API(OpenAI 또는 로컬 Ollama)를 호출해 답변을 주는 구조를 설계하되, `자바의 인터페이스`와 `팩토리 패턴`을 활용하여 런타임에 `AI 엔진을 dynamic하게 스위칭할 수 있는 소켓 서버` 기초 구현
